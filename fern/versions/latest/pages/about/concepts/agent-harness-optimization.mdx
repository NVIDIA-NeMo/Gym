**Goal**: Understand how environment scores guide agent-level improvements, what kinds of changes you can make without training, and when to escalate to model training.

## The Middle Ground

Evaluation tells you how well your agent performs. Training changes the model's weights to make it perform better. Between these two lies the work most practitioners spend most of their time on: **changing the harness, not the model**.

Agent optimization uses the same environments and scores as evaluation — but instead of reporting metrics, you use them to guide iterative changes to the agent harness. The model stays frozen; the harness evolves.

## The Evaluate-Improve Loop

The workflow is a single iterative cycle:

1. **Evaluate** — run the agent in environments, collect scores.
2. **Analyze** — identify failure patterns, compare against baselines.
3. **Improve** — make a harness-level change.
4. **Re-evaluate** — confirm gains hold, check for regressions.

This loop operates in minutes to hours — far faster than training, which operates in hours to days. Each iteration uses the same environment; only the harness changes between runs.

## What You Can Change

Agent optimization targets the harness — everything *around* the model. The main levers:

### System Prompts

The instructions that frame how the model approaches tasks. Changes range from minor (rephrasing, reordering instructions) to structural (conditional prompt assembly, few-shot examples, chain-of-thought scaffolding). Shorter, less prescriptive prompts tend to work better as models become more capable — except for safety-critical instructions, which should remain tight.

### Tools

The interfaces the agent uses to take action in the environment. Optimization includes adding or removing tools, rewriting tool descriptions so the model selects the right one, and managing tool catalog size — presenting too many tools at once degrades accuracy.

### Orchestration

The interaction logic that governs the agent's behavior across turns. This includes turn limits, retry strategies, self-correction loops, planning steps, and multi-role patterns (e.g. separate summarization and reasoning passes). Even without changing the model, restructuring the orchestration can dramatically change performance.

### Context Management

How the harness manages the model's context window across turns. In long multi-turn interactions, the growing context of accumulated tool results, observations, and prior reasoning must be pruned, summarized, or prioritized. Poor context management causes capable models to fail on tasks they could otherwise solve.

## Reading Scores for Optimization

Environment scores tell you more than pass/fail. Patterns in the scores guide what to change:

- **High variance across repeats on the same task** — the harness is fragile. Small changes in model output lead to different outcomes. Look at orchestration and error handling.
- **Consistent failures on a task category** — the harness may lack the right tools or prompts for that category. Look at tool coverage and system prompt specificity.
- **Correct reasoning but wrong final answer** — the model knows what to do but the harness loses the thread. Look at context management and output parsing.
- **Unnecessary tool calls or wasted turns** — the harness is over-scaffolding. Simplify the orchestration or tighten tool descriptions.

## When to Escalate to Training

Agent optimization has limits. When harness-level changes plateau — scores stop improving despite continued iteration — the bottleneck is the model itself. Signs:

- The model lacks domain knowledge that no prompt can provide.
- The model struggles with reasoning patterns that scaffolding can't compensate for.
- The model consistently ignores or misapplies instructions despite clear prompting.

At this point, the same environment that guided optimization becomes a training environment. The verifier scores that measured performance become the reward signal that drives model improvement. See [Training](/latest/about/concepts/training) for how this works.

## Agent Optimization and Training Are Complementary

Agent optimization and model training are not competing approaches — they compound. A better harness makes training more efficient (cleaner reward signal, fewer wasted rollouts). A better model makes the harness more effective (follows instructions more reliably, needs less scaffolding).

The most successful teams iterate on both: rapid harness changes to find the best interaction pattern, then training to bake capability into the model, then more harness iteration on the improved model.
