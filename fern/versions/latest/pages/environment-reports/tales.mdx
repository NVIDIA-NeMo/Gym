# TALES

`resources_servers/tales` wraps [TALES](https://github.com/microsoft/tale-suite)
(Text Adventure Learning Environment Suite, [arXiv:2504.14128](https://arxiv.org/abs/2504.14128)),
exposing five text-adventure frameworks through a Gymnasium-style resources
server. Rows select a task via `framework` / `task_no` / `split` / `seed` /
`max_episode_steps`; omitted fields fall back to `configs/tales.yaml`.

The integration pins the upstream `train_test_splits` branch. That is a
**mutable branch pin**; this documentation and the generated datasets were
verified at commit `ef349ba7cfdebf339e9aedcd09d89d6c917f86e5`.

## Task inventory

122 tasks total (`STRUCTURAL`, verified `MEASURED` at generation time):

| Framework | Tasks | `task_no` mapping | Index stability |
|---|---|---|---|
| `textworld` | 10 | TWCookingLevel1..10 — `task_no` 0..9 = difficulty 1..10 | stable |
| `textworld_express` | 16 | list literal, incl. SimonSays length (10/50/100) × memory × verbose variants | stable |
| `alfworld` | 12 | 6 task types × seen (`task_no` 0–5) / unseen (6–11) | stable |
| `scienceworld` | 30 | derived from the `scienceworld` library at import time | **unstable across library versions** (dependency unpinned) |
| `jericho` | 54 | `GAMES_INFOS` dict order (games.json) | stable only under the pin |

Built-in difficulty gradients: textworld difficulty 1–10, TWX sequence
lengths and memory/verbose variants, ALFWorld seen/unseen, ScienceWorld
per-task variations, Jericho per-game difficulty (walkthrough length mean
87.15 — the suite's hard tail).

<Warning>
**Seed semantics differ per family** (`STRUCTURAL`): `seed` selects *content*
(game file / variation / seed-pool index) for textworld, textworld_express,
alfworld, and scienceworld, but only stochasticity for jericho. In particular,
**alfworld rows must pin `seed=0`** — any other seed selects an upstream
*training* game file.
</Warning>

## Split enforcement

The row-level `split` field is a **silent no-op for 4 of 5 families**
(`app.py:71-74` falls back to the full task list; only scienceworld passes it
to the environment, `app.py:87-88`). Train/test hygiene is therefore enforced
**at the dataset row level** by the generator, which imports the upstream
split machinery (`JERICHO_TRAIN_GAMES`, 27 of 54 games) rather than
hand-writing index lists.

## Eval dataset tiers

Generated by `resources_servers/tales/scripts/generate_eval_datasets.py`
(deterministic; run it inside the server venv). All rows pin the paper-parity
system prompt, `seed`, `split="test"`, and `max_episode_steps=100` (10 for
smoke), and mirror ten slicing keys (family, framework, task_no, split, seed,
raw_id, capability, difficulty tier, contamination flag, tier membership) into
`responses_create_params.metadata`, which survives verbatim into rollout rows.
The longer `split_provenance` annotation is top-level only — recover it by
joining `_ng_task_index` back to the dataset row.

| File | Rows | Purpose |
|---|---|---|
| `data/smoke.jsonl` | 5 | wiring/health check only (1 per family, cap 10) — never for scoring |
| `data/eval_v0.jsonl` | 64 | curated tier: all tw/twx/alfworld, 12 category-balanced scienceworld, 14 spread jericho with anchors. Membership is **PROVISIONAL** until the k≥5 sweep classifies tasks saturated/discriminative/floored |
| `data/eval_full.jsonl` | 95 | the complete test-clean universe (122 − 27 jericho-train games, excluded as a named slice) |
| `data/paper_parity.jsonl` | 122 | full per-family populations incl. **all 54 jericho games** — matches the paper's eval population for harness-parity runs. **Pools jericho train+test; never use for training-adjacent evaluation** |

Jericho is the only contamination-flagged family (published walkthroughs;
canonical Infocom titles flagged separately). The other four families are
procedurally generated.

## Metric contract

The paper's reported metric is the **normalized running highscore**
(`PRIOR`, verified against upstream `benchmark.py`): per episode,
`highscore = max over steps of cumulative game score` (init 0), normalized by
`max_score`, averaged over games within a framework, then equal-weight
macro-averaged across the five frameworks. Success rate (`info["won"]`) is
reported alongside.

<Error>
**Never use the rollout `reward` field as a score.** The agent harness sums
per-step rewards, and for jericho the per-step "reward" is the *cumulative*
game score, so the sum is inflated (`STRUCTURAL`, `app.py:116-121`,
`gymnasium_agent/app.py:138`). Jericho is exactly where the paper's headline
"&lt;15% on human-written games" result lives — any jericho number that looks
paper-beating must be re-verified against the last-step score before being
believed. The repo's own example aggregate (`mean/reward = -19.8`) is a pooled
mean across incomparable reward scales and is meaningless as a benchmark
number.
</Error>

**Prerequisite server fix (required before baseline sweeps).** With current
code, no single rollout field reproduces the paper metric: `info["score"]` is
overwritten with the per-step value (`app.py:129-133`), destroying the clean
cumulative for scienceworld/twx (twx's surviving `total_score` is on a native
0–1 scale), and `aggregate_metrics` receives records without family identity.
The fix (~6 lines): read the upstream cumulative `game_score` *before* the
overwrite (uniform across all five families), track a running `highscore`,
and expose `game_score`, `highscore`, `normalized_highscore`, `won`, and
`framework` in step `info`, plus an `aggregate_metrics` override that groups
by `info["framework"]` and macro-averages `normalized_highscore`.

Until the fix lands, the fallback per-family extraction is (against current
code): textworld → `info["score"]`; jericho → `info["score"]` (never
`total_score`); alfworld → `info["score"]` (0/1); scienceworld →
`info["total_score"]`; textworld_express → `100 × info["total_score"]`
(native 0–1 scale). Terminal-score extraction understates non-monotonic games
(scienceworld can lose points; some jericho games deduct) — treat fallback
numbers as a lower bound there.

Secondary diagnostics: truncation rate, invalid-action rate,
steps-to-completion, tokens per episode.

## Reproduction protocol (harness parity)

Reproducing the paper's numbers through the Gym integration is the
harness-parity test — a discrepancy is an integration bug until proven
otherwise. Protocol (`PRIOR`, arXiv:2504.14128 §4, Tables 3/5):

- **Dataset**: `data/paper_parity.jsonl` (the 27-game jericho test split is a
  branch artifact the paper never used — `eval_full` is *not* paper-comparable
  on the jericho column or the macro-average).
- **Settings**: 100-step cap (**both** row `max_episode_steps` and the agent's
  `max_steps` in `tales.yaml` — the agent loop cap binds), admissible commands
  hidden, temperature 0, 5 repeats, the stock zero-shot prompt (byte-identical
  to the paper's).
- **Target**: a self-hosted exact paper model — primary
  `Llama-3.1-8B-Instruct`, expected per-framework normalized highscore
  `[tw 29.7±4.7, twx 50.3±2.9, alfworld 0.0, sw 15.7±0.9, jericho 2.3±0.1]`,
  average 13.9 (`PRIOR`, Table 3/5). API-endpoint models are proxies with no
  parity expectation.
- **Tolerance**: per-framework mean within ±max(5, 2σ_paper) points (our
  stated choice, derived from the paper's Table 5 variance). Reasoning-model
  rows are out of scope — the stock harness is single-shot.
