# Configuration

NeMo Gym reads model credentials from a top-level `env.yaml` file in the project root. This keeps secrets out of your YAML configs and out of version control (`env.yaml` is gitignored).

## env.yaml

Create `env.yaml` in the project root:

```yaml
policy_base_url: https://api.openai.com/v1
policy_api_key: sk-your-actual-key-here
policy_model_name: gpt-4.1-2025-04-14
```

Three fields, all required:

- **`policy_base_url`** — the OpenAI-compatible endpoint
- **`policy_api_key`** — your key for that endpoint
- **`policy_model_name`** — the model to call

## Other Providers

The same three fields work for any OpenAI-compatible endpoint:

<Tabs>
<Tab title="Azure OpenAI">

```yaml
policy_base_url: https://<your-resource>.openai.azure.com/openai/v1
policy_api_key: <your-azure-key>
policy_model_name: gpt-4.1-2025-04-14
```

</Tab>
<Tab title="vLLM (self-hosted)">

```yaml
policy_base_url: http://localhost:8000/v1
policy_api_key: not-needed
policy_model_name: Qwen/Qwen3-4B-Instruct-2507
```

The model must support function calling for tool-using agents.

</Tab>
</Tabs>

<Tip>
**Why `gpt-4.1-2025-04-14` for the Quickstart?** It has low latency (no reasoning step) and reliable function-calling support, so the Quickstart runs in seconds without configuration surprises. Swap in any model later.
</Tip>

## CLI Overrides

You can override any field on the command line with `+key=value`:

```bash
ng_run "+config_paths=[...]" ++policy_model_name=gpt-4o-mini
```

Use this for one-off runs without editing `env.yaml`.

You're set. Run the [Quickstart](/latest/get-started/quickstart).
