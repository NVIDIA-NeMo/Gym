Train Models picks up after a NeMo Gym environment exists. The same environment that scored a model in [Evaluate Models](/latest/evaluate) produces the rollouts you train against. Pick the workflow first, then the framework.

<Tip>
Refer to [Training Approaches](/latest/about/concepts/training-approaches) for a refresher on when GRPO beats SFT beats DPO.
</Tip>

## Pick a Workflow

Start with the question you're answering, not the framework you have on hand. Each workflow lists the framework pages that implement it today.

| If you want to... | Workflow | Framework pages |
|---|---|---|
| Improve a model on a verifiable task using on-policy RL | **GRPO (online RL)** | [NeMo RL](/latest/train-models/nemo-rl-grpo), [Unsloth](/latest/train-models/unsloth), [OpenRLHF](https://github.com/OpenRLHF/OpenRLHF/blob/main/examples/python/agent_func_nemogym_executor.py) |
| Train on multiple training environments at once (multi-verifier) | **Multi-environment GRPO** | [Multi-Environment Training](/latest/train-models/multi-environment-training) |
| Fine-tune a base model on a successful-rollout dataset (no environment in the loop) | **SFT (offline)** | [Offline Training (SFT/DPO)](/latest/train-models/offline-training-w-rollouts) |
| Optimize against preference pairs collected from rollouts | **DPO (offline)** | [Offline Training (SFT/DPO)](/latest/train-models/offline-training-w-rollouts) |
| Reproduce a shipped model recipe end-to-end | **Recipe** | [Nemotron 3 Nano](/latest/train-models/recipes/nemotron-3-nano), [Nemotron 3 Super](/latest/train-models/recipes/nemotron-3-super) |

## What You Choose Inside Each Workflow

Independent of framework, every training run picks four things:

- **Environment** — the resources server (the verifier) that scores rollouts.
- **Agent** — the harness that orchestrates the rollout. See [Choose an Agent](/latest/get-started/choose-an-agent).
- **Model** — the policy under training; an OpenAI-compatible endpoint or a managed local vLLM.
- **Trainer** — the framework that consumes Gym rollouts and updates the model.
