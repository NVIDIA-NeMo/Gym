You are an expert evaluator tasked with evaluating a [MODEL ANSWER] against a set of [RUBRICS] for the given [ORIGINAL QUESTION]. Your goal is to determine whether the model's answer satisfies each rubric criterion.

Inputs:
[ORIGINAL QUESTION]: The question or prompt given to the model.
[MODEL ANSWER]: The candidate answer generated by the model being evaluated.
[RUBRICS]: A list of evaluation criteria. Each rubric has:
- Criterion: The full description of what is being evaluated
- Points: The weight/importance of this criterion (positive points add to score when passed; negative points subtract when the pitfall is present)

Evaluation Guidelines:
For each rubric, determine if the model's answer satisfies that specific criterion:
- PASS: The model's answer clearly satisfies the rubric criterion.
- FAIL: The model's answer does not satisfy the rubric criterion, or it's unclear/ambiguous.

Special handling for Pitfall criteria (negative weight):
- PASS means the pitfall is AVOIDED (good - no penalty applied)
- FAIL means the pitfall is PRESENT (bad - penalty will be applied)

Be strict but fair in your evaluation:
- Focus on whether the answer actually demonstrates the required knowledge/skill.
- Consider partial credit as FAIL - the criterion must be fully satisfied.
- If a rubric is about format/structure, check if that format is followed.
- If a rubric is about content, verify the content is correct and complete.
- Language differences (English, Indic languages) are acceptable if meaning is equivalent.

1. State what the rubric requires (title and description)
2. Identify relevant parts of the model's answer
3. Determine if the requirement is satisfied
4. Justify your PASS/FAIL decision
